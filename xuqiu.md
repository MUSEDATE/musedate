二、 详细页面规划 (Page by Page)
Page 1: Hero 屏 — 愿景与主张
- 核心目标： 第一眼建立“高级、严肃、AI驱动”的品牌认知。传达产品的核心愿景。
- 建议展示内容： 干净的背景，极简的 Typography，配合一个具有科技感与情感温度融合的主视觉（例如抽象的光晕、两个靠近的粒子球体，或极简的手机界面 Mockup）。
- 建议视觉风格： 深色或极浅的干净背景，文字排版大气留白，主色调（紫/蓝绿）作为点缀。
- 建议动画/交互： 页面加载时，主标题和副标题缓慢淡入（Fade In Up），主视觉带有非常轻微的呼吸感或视差效果。
- 与下一页衔接： 向下滚动时，当前元素淡出，背景平滑过渡。
建议文案：
[主标题] 在世界波动时，寻得一份确定的稳定。
[副标题] Muse 觅Ta —— AI 驱动的严肃恋爱平台。让“随时遇见合适的人”成为可能，只在你真正准备好时，开启真实的连接。
[按钮] 开启寻找之旅 / 了解更多
Page 2: 痛点与破局 — Why MuseDates
- 核心目标： 引起共鸣，指出当前 Dating 软件的低效与意图不匹配。
- 建议展示内容： 左右对比布局，左侧列出痛点，右侧展示 Muse 的破局点。可以使用图文并茂的卡片。
- 建议视觉风格： 清晰的信息层级，痛点使用暗色调或灰色，解决方案使用品牌亮色。
- 建议动画/交互： 滚动到该区域时，卡片依次滑入（Staggered reveal）。
建议文案：
[引言] 为什么寻找对的人，总是充满疲惫？
[痛点卡片 1] 意图混杂：想认真恋爱，却总遇到只想聊天的过客。
[痛点卡片 2] 标签单薄：仅凭几张照片和干瘪的标签，无法看透真实的灵魂。
[破局点主张] Muse 觅Ta，重新定义连接。
[方案描述] 我们通过智能意图识别过滤噪音，基于多维 Persona 还原真实的你，实现全局最优匹配，拒绝低效等待。
Page 3: 核心机制 1 — 意图识别与多维匹配
- 核心目标： 展现 AI 在匹配上的硬核实力，强调“懂你”。
- 建议展示内容： 一个动态的数据雷达图或维度的拆解可视化，展示“颜值偏好、价值观、生活方式”等维度的精准匹配。
- 建议视觉风格： 科技感的数据可视化，配合简洁的界面截图（如匹配度 98% 的资料卡局部）。
- 建议动画/交互： 随着滚动，雷达图各项指标亮起，两组数据逐渐重合。
建议文案：
[主标题] 不止看脸，更看懂你。
[正文] Muse 独创多维 Persona Embedding 匹配机制。从客观背景到内在特质（家庭观念、幽默感、生活方式），再到审美偏好的深度学习。我们不仅知道你喜欢什么样的外表，更明白你需要什么样的灵魂共鸣。
Page 4: 核心机制 2 — 小me / 数字人助手
- 核心目标： 介绍差异化最大亮点：数字人预聊与陪伴。
- 建议展示内容： 手机端聊天界面的 Mockup，展示用户与自己“小me”对话更新偏好的过程，或者他人与你的“小me”预聊的场景。
- 建议视觉风格： 模拟真实的对话气泡，突出 AI 的智能与亲和力。
- 建议动画/交互： 模拟打字效果，聊天气泡逐个弹出。
建议文案：

[正文] 遇见“小me”。它是你的专属匹配助手，在日常对话中不断学习你的喜好，为你精准调优推荐模型。它也是你的社交前哨，支持让心仪对象与你的数字分身预聊，打破破冰尴尬，预判契合度。

建议文案：

[副标题] 我们不为盈利设置障碍，而是为真诚设立护城河。

[图表建议] 可做 3 列对比：传统相亲 / 普通 Dating App / Muse。对比维度建议包括：金钱成本、时间成本、信息有效度、意图清晰度、匹配效率、进入真实聊天前的判断能力。
[图表结论文案] Muse 不是“最便宜”的方式，而是“综合性价比更高”的方式：用更低的试错成本，换更高质量的遇见。

三、 整体视觉风格与动效建议
1. 整体视觉风格 (Art Direction)
- 品牌气质： 克制、高级、真诚、略带未来感但不冰冷。这是一款“懂你”的严肃交友软件，而不是喧闹的社交广场。
- 配色建议：
  - 背景：#FCFCFC (极浅灰/白) 或 #F7F7FA，保持干净。
  - 品牌主色：#6B46C1 (一种稳重且带有神秘感的紫色) 或 #2B6CB0 (沉稳的蓝色)。用于按钮、重点文字、强调线。
  - 文字色：#2D3748 (深灰)，避免纯黑的压迫感。
- 字体建议： 无衬线体，如 Noto Sans SC, PingFang SC，排版注重行距和字间距的呼吸感。
- 素材建议：
  - UI Mockup：使用简洁的 3D 手机外壳，内部展示真实的 Muse 界面（如雷达图、聊天界面）。
  - 插画/3D：可以少量使用高级质感的 3D 玻璃拟物（Glassmorphism）元素来表达“AI 维度”和“透明度”。避免廉价的矢量小人插画。
2. 整体动效节奏 (Motion & Interaction)
- 滚动叙事 (Scroll-telling)： 页面不宜过长，每一屏占据合适的视口比例。随着滚动，核心信息平滑淡入（Fade In Up）。
- 视差效果 (Parallax)： 背景元素或手机 Mockup 与文字滚动的速度产生微差，增加空间感。
- 微交互： 例如在“多维匹配”展示区，鼠标悬停时展示具体的维度得分；在“小me”展示区，模拟真实的打字输入动效。
四、 给网页设计/开发同学的落地提示
1. 优先级策略： 如果时间紧迫，优先打磨 Page 1 (Hero 首屏)、Page 3&4 (意图匹配与小me核心差异化)、以及 CTA 屏。这几部分是转化和传递价值的关键。
2. 关于视频素材： 如果来不及制作 APP 使用的录屏视频，可以使用 Lottie 动画 模拟界面交互，或者使用高质量的 静态 UI 截图 + CSS 渐入动效 来替代。重点是传达“清爽、高级”的界面感觉。
3. 手机 Mockup 组织： 在展示功能（如绝配广场、聊天）时，不要干巴巴地放截图。把截图嵌在半透明或极简的手机壳样机中，适当倾斜或漂浮，更有产品质感。
4. 文案落地： 文档中的文案已尽量去除了“土味营销感”，排版时请注意留白，字号对比要拉开，不要挤成一团。标题要大，正文颜色要柔和。
